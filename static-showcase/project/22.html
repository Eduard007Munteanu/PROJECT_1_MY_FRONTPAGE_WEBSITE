Building the Internet of Things with P2P and Cloud Computing : AirBrush Project

Airbrush is a multiplayer browser drawing game where players use phone motion sensors to draw in the air, while others guess the drawing on a shared canvas. The project explores WebRTC-based peer-to-peer communication and IoT-style motion interaction.

Project Description

Airbrush explores how a smartphone can be used as a motion-based drawing controller in a browser game. One player acts as the drawer, using accelerometer and gyroscope input from their phone to create brushstrokes on a shared canvas, while the other players guess the word being drawn.

The main technical focus was building a real-time multiplayer system without relying on a traditional central game server for all communication. The project uses WebRTC data channels for peer-to-peer synchronization, while a signaling server is only used to help peers discover each other and establish connections. This made the project relevant for understanding decentralized communication, NAT traversal, synchronization, and real-time state sharing between browsers.

The prototype also includes game-state management such as player roles, chat, scoring, round progression, word hints, and canvas synchronization. A key challenge was balancing responsiveness with noisy motion-sensor data, since phone movement had to be translated into smooth and understandable drawing input. Overall, the project combines IoT sensor interaction, browser-based game development, and peer-to-peer networking into one experimental drawing game.
