Mere konkret, jeg har brugt disse programmeringssprog til følgende:

Java har jeg brugt til bl.a. backend for en hjemmeside jeg lavede I HCI og en i Eksperimentel systemudvikling. Sproget blev også brugt til at lave en arkitekturelt korrekt struktureret software, et computerspil, som nøjagtigt følgte principer som TDD (Test Driven Development).

Python brugte jeg til matematiske baserede fag som f.eks. Numærisk Linær Algebra, Machine Learning, Data Mining, Introduction til Matematik og optimering, med libraries som f.eks MatPlotLib, Numpy, Sklearn, Sage, etc. Python brugte jeg også til en P2P distribueret hjemmeside som backend til min Cloud Server som blev brugt til hosting af en signaling server for peers (clients i P2P systemet)

HTML/JavaWScript/CSS blev brugt til frontend for de hjemmeside jeg brugte Java og Python som backend.

Jeg har brugt C lidt til håndtering af små opgaver basered på programmering med multiple threads, sammen med mere graunleret kontrol over brugen af memory. Flerer threads mener også at jeg håndterede udfordringer relatered til f.eks. Race Conditions og Deadlock. C, sammen med ARM og X86 blev brugt sammen for at have granuleret kontrol over memory brugen uden brug af garbage collectors. Dog jeg vil nævne at jeg har kun lidt erfaring med disse.

ARM + Intel X86 Assembly blev brugt sammen med C. Noget der er lidt højere niveau programmeringssprog end ARM med lavere niveau programmeringssprog end C er LLVM. Jeg brugte LLVM til at lave et selvdefineret programmeringssprog kaldet Dolphin (project fra faget "Oversættelse" ).

C# med Unity blev brugt til at lave både VR og AR simulationsspil.

Ocaml blev brugt til at lave Disstriburede Systemer, både Asynchronous og Synchrnous systemer.

Scala blev brugt til Programmeringssprog faget, hvor vi lavede en mini compiler. Vi brugte funktionel programmering, recursive calls med og uden tailroad.

SQL og MySQL blev brugt for backend til diverse hjemmesider.
